# 贡献指南

欢迎为 **gao-prompts** 提交新的提示词、修订或使用反馈。

## 新增提示词

1. 在 `prompts/` 目录下新建一个 markdown 文件
2. 文件名使用 **英文 kebab-case**,清晰表达"输入 → 输出"语义
   - ✅ `claude-session-to-subagent.md`
   - ✅ `pr-diff-to-changelog.md`
   - ❌ `prompt1.md` / `通用提示词.md` / `MyPrompt.md`
3. 文件开头使用一个 `#` 一级标题作为提示词名称,紧接一段简短摘要说明:
   - 它做什么
   - 输入是什么(需要用户准备什么前置数据)
   - 输出是什么
   - 是否有特殊使用前提(如需要新开会话、分多轮执行等)
4. 在根目录 `README.md` 的"提示词索引"表格中追加一行

## 提示词写作建议

- **可执行优先**:每条规则应能被 LLM 直接照做,避免"应该尽量"、"通常情况下"等软词
- **证据驱动**:涉及具体规则的地方,带数字阈值或字面判断条件
- **结构清晰**:多轮提示词请显式分章节,标注每轮的输入依赖
- **隐私安全**:示例中不要包含真实用户数据;若引用了 session,请脱敏

## 提交流程

1. Fork 本仓库
2. 创建分支 `feat/<short-name>` 或 `fix/<short-name>`
3. Commit message 推荐遵循 Conventional Commits(`feat:` / `fix:` / `docs:` 等)
4. 提交 Pull Request,在描述中说明:
   - 这个提示词解决什么问题
   - 你是否实际用过、效果如何
   - 是否有已知局限

## 反馈与讨论

- 使用中遇到的问题、建议、改进点,欢迎提 Issue
- 如果你基于现有提示词产出了有效的 subagent / 工作流,欢迎在 Discussion 区分享(注意脱敏)
